# Tabs

## Base

This is the most basic version of the component and its default appearance.

<Canvas of={Stories.Basic} />

## Modifiers

### Is contained

When we want to differentiate the tabs from other blocks (e.g. when we use tabs
as secondary navigation, below te AppBar) we can get a more emphasized styles,
adding the modifier `tabs__container--contained` to the container.

<Canvas of={Stories.Contained} />

### Is primary

To get a primary color scheme Tabs we only have to add the modifier `tabs__marker--primary` to
the tabs marker (the line below the active tab).

<Canvas of={Stories.Primary} />

### Wide

To fill the whole space available of the parent you have to add to each `TabsItem` the modifier `tabs__item--wide`.

<Canvas of={Stories.Wide} />

### Sizes

We can get the different sizes adding to the `tabs__item` and `tabs__link` elements
the modifiers `tabs__item--sm`, `tabs__item--md`... etc.

<Canvas of={Stories.Sizes} />

## Elements

### TabsItem

This is the most basic version of the component and its default appearance.

<Canvas of={Stories.TabsItemBase} />

#### With icon

An icon can be added at the beginning to visually reinforce the Tab meaning. It
requires an 'xs' margin between icon and text (`.margin-cmp--right-xs`).

<Canvas of={Stories.TabsItemIcon} />

#### With remove button

A 'remove' IconButton can be added at the end to close the tab. It requires an
'xs' margin between text and button (`.margin-cmp--left-xs`).

<Canvas of={Stories.TabsItemRemove} />
